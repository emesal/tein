# `#[tein_const]` support in `#[tein_module]`

issue: #59

## summary

add `#[tein_const]` item annotation to `#[tein_module]`, allowing rust constants
to be exposed as scheme bindings via pure scheme `(define ...)` emission in the
generated VFS `.scm` file.

## data model

```rust
struct ConstInfo {
    scheme_name: String,    // e.g. "uuid-nil"
    scheme_literal: String, // e.g. "\"00000000-0000-0000-0000-000000000000\""
}
```

added to `ModuleInfo` as `consts: Vec<ConstInfo>`.

## naming

- `SCREAMING_SNAKE` → `kebab-case`, bare (no module prefix)
- `UUID_NIL` → `uuid-nil`
- override via `#[tein_const(name = "custom-name")]` using existing `extract_name_override`

## supported types (macro-time literal extraction)

| rust type | scheme output |
|-----------|--------------|
| `&str` / string literal | `"..."` |
| integer literal | fixnum |
| float literal | flonum |
| `true` / `false` | `#t` / `#f` |

## codegen

**`generate_vfs_scm`**: emits `(define name value)` forms. no extern C wrapper
needed — consts are pure scheme.

**`generate_vfs_sld`**: adds const scheme names to the export list.

**`generate_register_fn`**: unchanged. consts live entirely in VFS scheme files.

**`parse_module_info`**: new `Item::Const` arm with `#[tein_const]` attribute.
const item passes through to `clean_items` (the rust const remains in the module).

## example

```rust
#[tein_module("uuid")]
mod uuid_impl {
    #[tein_const]
    const UUID_NIL: &str = "00000000-0000-0000-0000-000000000000";
}
```

generates in `.scm`:
```scheme
;; (tein uuid) — generated by #[tein_module]
(define uuid-nil "00000000-0000-0000-0000-000000000000")
```

and exports `uuid-nil` in `.sld`.
