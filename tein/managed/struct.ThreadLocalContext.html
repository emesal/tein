Expand description

A managed Scheme context on a dedicated thread.

Wraps a Context running on a dedicated thread. Requests are proxied over channels, making this type Send + Sync and safe to share across threads via Arc.

§Modes

  • Persistent: context lives across evaluations. State accumulates. reset() tears down and rebuilds from the init closure.
  • Fresh: context is rebuilt before every evaluation. No state leakage. reset() is a no-op.

§examples

use tein::{Context, Value};

let ctx = Context::builder()
    .step_limit(1_000_000)
    .build_managed(|ctx| {
        ctx.evaluate("(define x 42)")?;
        Ok(())
    })?;

let result = ctx.evaluate("(+ x 1)")?;
assert_eq!(result, Value::Integer(43));

pub fn reset(&self) -> Result<()>

Rebuild the context from the init closure.

In persistent mode, tears down the current context and rebuilds from the stored builder config + init closure. Live foreign objects are dropped. In fresh mode, this is a no-op (already rebuilds each call).
